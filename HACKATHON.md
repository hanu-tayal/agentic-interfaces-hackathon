Source: https://seattle.aitinkerers.org/hackathons/h_x_n9rN2gpzc

- Name: Generative UI Global Hackathon: Agentic Interfaces
- Host: AI Tinkerers Seattle
- Date: Saturday, May 9, 2026
- Build window: 1:00 PM-5:00 PM PDT
- Show and tell: 5:00 PM-5:45 PM PDT
- Submission window: 5:45 PM-6:00 PM PDT
- Submission deadline: 6:00 PM PDT
- Global results: Friday, May 15, 2026

## Core Requirement

Build an AI application where the agent generates interactive UI at runtime.

The project should not be a chatbot with a static interface. The test from the handbook is:

> Would this have been impossible with a chat interface?

## Acceptable Technology Directions

Use one or more of:

- A2UI: open protocol for agents to send interactive UI
- AG-UI: transport layer for agent-to-frontend communication
- CopilotKit: React framework for embedding AI copilots
- MCP Apps: Model Context Protocol applications rendered as live UI
- Other open-source tools, public APIs, or sponsor technologies

## Track Options

- Kill the Dashboard: generate the exact visualization, form, or control surface needed in the moment.
- The Copilot That Ships: render interactive UI for users to confirm, tweak, and execute inline.
- Agent App Store: use MCP-powered apps where agents discover, compose, and present multi-tool experiences.
- No Designer, No Problem: generate the full user-facing interface at runtime.

## Submission Requirements

Final submission must include:

- Project name
- One-sentence pitch
- Short description of what was built
- Explanation of why it is generative UI, not a chatbot
- Public GitHub repo link
- 2-3 minute demo video link
- Protocols used: A2UI, AG-UI, CopilotKit, MCP Apps, or other
- Team member names and roles

## Repo Requirements

- This repo is currently private because it was created as a private working repo.
- Before final submission, make the repo public or provide a public fork/link.
- Be transparent about any pre-existing code or starter kit used.
- Keep enough setup instructions that judges can understand and run the project.
- Include demo-ready code, not slides or a pitch-only artifact.

## Judging Focus

Projects are evaluated globally after submissions close. Evaluation focuses on:

- Working code
- Originality
- Effective use of generative UI

## Demo Requirements

- Demo length: 2-3 minutes
- Show working code only
- One person demos
- Whole team can answer questions

## Rules That Affect This Repo

- Teams can have up to 4 people.
- Solo teams are allowed.
- In-person attendance at a participating city is required.
- The team owns IP created during the event.
- Pre-existing code is allowed, but must be disclosed.
- Open-source tools, public APIs, and sponsor technologies are allowed.
